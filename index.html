Notación Asintótica

La notación Big-O nos ayuda a entender cómo crece el tiempo de ejecución de un algoritmo.

Notación Crecimiento
O(n) Lineal
O(n²) Cuadrático
O(n³) Cúbico
O(2ⁿ) Exponencial

Algoritmo A - Multiplicación de matrices

for i = 1 hasta n
   for j = 1 hasta n
      for k = 1 hasta n
         operación

Proceso con n = 5

Paso 1: El ciclo i se repite n veces.

Paso 2: Dentro de i se ejecuta el ciclo j n veces.

Paso 3: Dentro de j se ejecuta el ciclo k n veces.

Ciclo Repeticiones
i 5
j 5
k 5

Total de operaciones:

5 × 5 × 5 = 125

Complejidad final → O(n³)

Algoritmo B

for i = 1 hasta n
   for j = i+1 hasta n
      for k = 1 hasta j
         operación

Proceso con n = 5

i Valores de j Iteraciones de k
1 2,3,4,5 1..5
2 3,4,5 1..5
3 4,5 1..5
4 5 1..5

El crecimiento sigue siendo cúbico.

Complejidad → O(n³)

Algoritmo C

for i = 1 hasta n
   si i es impar
      for j = i hasta n
   si no
      for j = 1 hasta i

Proceso con n = 5

i Tipo Iteraciones
1 Impar 5
2 Par 2
3 Impar 3
4 Par 4
5 Impar 1

La suma de operaciones crece aproximadamente como n².

Complejidad → O(n²)

Algoritmo D - Recursivo

funcion recursiva(n)

if n <= 1
 return 1
else
 return recursiva(n-1) + recursiva(n-1)

Proceso con n = 5

Nivel Llamadas
1 1
2 2
3 4
4 8
5 16

Cada llamada genera dos nuevas llamadas.

Complejidad → O(2ⁿ)
